# Agent 优化清单

> 当前版本先跑通，这些改进点留到后续迭代。

## 🔴 高优先（显著提升体验）

### 1. 记忆压缩 — Summary Buffer

**现状**：对话超长时，简单丢弃旧消息（滑动窗口）

**问题**：早期对话包含关键信息（用户目标、数据字段含义），丢了 LLM 就"失忆"

**方案**：
```
旧消息前半段 → 调用 LLM 压缩成一条摘要 → 替换原文
新消息后半段 → 保留原文不动
```

**参考**：Claude 的上下文压缩机制

**实现要点**：
- 消息超过阈值时触发压缩
- 用一个小 LLM 请求做摘要（`llm.chat.completions.create`）
- 摘要格式：`{"role": "system", "content": "[历史摘要] ..."}`

---

### 2. 流式输出（SSE）

**现状**：Agent 全部执行完才返回结果，用户干等

**方案**：用 Server-Sent Events 实时推送每个 ReAct 步骤

```
前端收到事件流：
  event: thought     → "正在预览数据结构..."
  event: action      → "调用 data_preview"
  event: observation → "数据共 1000 行..."
  event: done        → "分析完成"
```

---

### 3. 工具错误自修复

**现状**：代码执行报错时，错误信息返回给 LLM，LLM 自行修复（基础版）

**增强**：
- 错误分类（语法错误 / 运行时错误 / 超时 / 沙箱不可达）
- 常见错误的自动修复提示（如"pandas 未导入 → 自动补 import pandas"）
- 重试计数（同类型错误超过 3 次就放弃）

---

## 🟡 中优先（功能完善）

### 4. 图表渲染工具

**现状**：LLM 生成 ECharts 配置 JSON，前端渲染

**增强**：
- 沙箱预装 matplotlib，生成预览图
- Agent 直接输出 `echart_option` 结构化 JSON
- 支持图表类型推荐（柱状/折线/饼图/散点）

### 5. 数据缓存

**现状**：每次读文件都从磁盘重新读

**方案**：首次读取后缓存 DataFrame 到 Redis，后续操作直接用缓存

### 6. 多轮对话深度记忆

**现状**：只有当前会话的历史

**方案**：用户偏好记忆——"这个用户喜欢饼图"、"上次分析过销售数据"
